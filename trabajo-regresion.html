Ajuste de una recta, usando el algoritmo de gradiente

En esta parte, vamos a ajustar una recta para intentar explicar la nota media en funcin del ranking del alumno en la PAU. Aunque, sea mucho (muchsimo!) ms sencillo usar la instruccin lm de R para realizar el ajuste, implementaremos el algoritmo del gradiente para ir encontrando el mnimo de la funcin coste.

Lo haremos en varias etapas

Implementacin de la funcin coste.

Recordar que la funcin coste es (ver transparencias): \[J(\theta)=\frac{1}{n}\sum_{i=1}^n\left(y_i-x_{i\bullet}^T\theta\right)^2=\frac{1}{n}\lvert\lvert \mathbf{y}-\mathbf{X}\theta\rvert\rvert^2=\frac{1}{n}\left(\mathbf{y}-\mathbf{X}\theta\right)^T\cdot \left(\mathbf{y}-\mathbf{X}\theta\right),\] donde \(\mathbf{y}\) es el vector que contiene todas las observaciones de la variable respuesta, y la matriz \(\mathbf{X}\) la matriz de diseo.

Definir una variable y que contenga los valores de notamedia, y la matrix X que sea la matriz de diseo asociada a la frmula:

\[notamedia=\theta_0+\theta_1PAU\_CALIFICACION\_NOTA\_MATERIA\]

# Completar aqu
y <- GradosTelecoSimple$notamedia
notamedia <- as.matrix(cbind(1,GradosTelecoSimple$PAU_CALIFICACION_NOTA_MATERIA))

Vamos ahora a implementar la funcin coste que llamaremos J as como el gradiente de J, que llamaremos gradJ. De paso, aprenderemos a definir una funcin en R.

J <- function(theta)
  {
  # tenes que calcular el coste aqu
  coste <- (1/length(y)) * t(y-notamedia%*%theta) %*% (y-notamedia%*%theta)

  # fin completar
  return(coste)}

El gradiente de la funcin coste se puede escribir de manera compacta (ver transparencias) \[\nabla J(\theta)=\frac{2}{n} \mathbf{X}^T\cdot \left(\mathbf{X}\theta-y\right).\]

gradJ <- function(theta)
  {
  ## completar aqu para devolver el vector gradiente
  gradiente <- (2/length(y))*t(notamedia)%*%(notamedia%*%theta-y)
  
  return(gradiente)
  }

Implementacin del algoritmo del gradiente

Una vez que tenemos implementada la funcin de coste, podemos escribir el cdigo para el algoritmo iterativo del gradiente.

Empezamos por fijar un valor inicial de theta, el valor de \(\alpha\), el learning rate y tambin el nmero mximo de iteraciones que autorizaremos para el algoritmo.

# nada que completar
thetainicial <- c(1, 1)
alpha <- 0.001
maxiter <- 3000

Recordad que la etapa de actualizacin es: \[\theta\leftarrow \theta-\alpha \nabla J(\theta).\]

Usando un bucle (la instruccin for en R, podis buscar en la ayuda), implementad el algoritmo del gradiente usando esta frmula:

## Completar aqu con un bucle el algoritmo del gradiente
theta=thetainicial
for (i in 1:maxiter)
  {
    theta <- theta - alpha * gradJ(theta)
    
  }
